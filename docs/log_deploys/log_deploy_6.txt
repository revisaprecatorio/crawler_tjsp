🎯 Descobrimos o Problema!
Você está tentando executar o crawler no host (servidor), mas as dependências (Selenium) estão dentro do container.

O container está em loop de restart. Vamos aguardar e então entrar nele:

bash
# 1. Aguardar 15 segundos para o container estabilizar
sleep 15

# 2. Verificar se está rodando
docker compose ps

# 3. Entrar no container
docker exec -it tjsp_worker_1 bash

# 4. Dentro do container, executar o crawler
python crawler_full.py \
  --doc 0077044-50.2023.8.26.0500 \
  --abrir-autos \
  --baixar-pdf \
  --turbo-download \
  --download-dir /app/downloads/07620857893 \

bash
# 1. Parar o container
docker compose down

# Mudar de: restart: unless-stopped
# Para: restart: "no"

# 3. Subir novamente
